files

Include specific file list. Convenient to use if there is one ore a couple of application entry points.

Type:

string[]

Default:

none

Hints:

  • All files are not included to compilation will be fully ignored by compiler.

    Example project file structure:

    src
├── bye.ts
├── greeter.ts
└── index.ts

    Content of tsconfig.json:

    {
  "compilerOptions": {
    "outDir": "./dist"
  },
  "files": [
    "src/index.ts"
  ]
}

    File index.ts doesn't use either bye.ts nor greeter.ts:

    console.log('index.ts')

    Only index.ts will be compiled. Any other files will be totally skipped. Even though they contain syntax errors. Fot instance greeter.ts file does:

    greeter.ts:

    function sayHello(name: string): {
                             // missed returning value after :
  console.log('Hello ' + name)
}

    The result:

    dist
└── index.js

  • All files are imported inside of specified files will be compiled too.

  • Not an option if you are using additional tools for build process like bundlers. Use include instead.

  • An error occurs if any of the files can't be found.

Relations:

  • related include
  • modified by include

include

Bunch file include using glob patterns.

Type:

string[]

Default:

if files is defined then [] else ["**/*"]

Hints:

  • It's just shortcut for files. Technically, this options are used to fill files options.

  • Include all files even though they aren't used in the program.

  • If a glob pattern doesn't include a file extension, then it will be defaults: .ts, .tsx, and .d.ts.

  • If allowJs are enabled, it adds .js and .jsx to defaults of glob pattern.

  • Right option to work with different bundlers.

Relations:

  • modifies files
  • related files
  • modified by exclude

exclude

Exclude some glob patterns which was already added by include.

Type:

string[]

Default:

["node_modules", "bower_components", "jspm_packages"], if outDir is defined then adds value of outDir, if declarationDir is defined then adds value of declarationDir

Hints:

  • Can only exclude file paths defined in include.

  • Can not have impact to files directive.

  • Can not exclude file if it's imported in other file which is included.

  • Once you declared exclude explicitly all the default values should be added manually if it's necessary.

Relations:

  • modifies include

target

Set the ECMAScript version for emitted JavaScript.

Type:

enum

Default:

ES3

Relations:

  • changes module
  • changes lib

module

Sets the module system for the program.

Type:

enum

Default:

if target is equal ES3 then CommonJS, ES5 then CommonJS, ES6 then ES6, ES2015 then ES2015 else ESNext.

Relations:

  • changes moduleResolution
  • changed by target

moduleResolution

Specify how TypeScript looks up a file from a given module specifier.

Type:

enum

Default:

if module is equal AMD then Classic, UMD then Classic, System then Classic else Node.

Hints:

  • In most cases you won't need to use Classic in modern code.

  • It makes sense to always set it up to Node manually.

Relations:

  • changed by module

lib

Specify what bundled library declaration files will be included.

Type:

enum[]

Default:

none

Hints:

  • It's able to include whole EcmaScript standards or its some parts.

  • It adds only type definitions, not polyfills. If you add some lib from higher standard to lower you have to manually add polyfills for it.

  • Once you declared lib explicitly all the default values should be added manually if it's necessary.

Relations:

  • changed by target
